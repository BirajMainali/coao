# Research Policy

## Purpose

Reduce uncertainty before making decisions.

## Principles

- Understand the problem before searching for solutions.
- Prefer existing organizational knowledge before external sources.
- Verify important information using authoritative sources.
- Do not assume when evidence can be gathered.
- Record important findings for future reuse.

## Research Order

For every task:

1. Current mission artifacts
2. Existing project code
3. Organizational knowledge
4. Previous architectural decisions (ADR)
5. Official documentation
6. Trusted external sources

## Expectations

Before asking for clarification:

- Search existing implementation.
- Search documentation.
- Search organizational knowledge.
- Identify exactly what remains unknown.

Questions should be specific and based on evidence.

## Deliverables

Research should produce:

- Findings
- Evidence
- Assumptions
- Risks
- Recommendations
- Confidence level

## Anti-patterns

Avoid:

- Guessing
- Copying solutions without understanding
- Searching only one source
- Ignoring previous decisions
- Research without summarizing findings
