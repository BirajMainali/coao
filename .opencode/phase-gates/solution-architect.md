# SA Completion Gate

Before marking work complete, verify each item below.

## Design

- [ ] Business requirements are fully understood
- [ ] Multiple solution options were explored
- [ ] Trade-offs documented
- [ ] Simplest viable approach chosen
- [ ] Architecture aligns with existing system
- [ ] Reuses existing capabilities where possible
- [ ] No unnecessary complexity introduced

## Evaluation

- [ ] Business alignment checked
- [ ] Maintainability checked
- [ ] Consistency checked
- [ ] Complexity justified
- [ ] Performance sufficient
- [ ] Scalability considered
- [ ] Reliability/error handling defined
- [ ] Security risks evaluated
- [ ] Operational impact (deploy, monitor, recover) considered
- [ ] Cost evaluated (dev, maintenance, infra)

## Artifacts

- [ ] context.md Design Overview section written
- [ ] context.md Workflow section updated
- [ ] context.md Current Status updated
- [ ] context.md Open Questions & Risks updated
- [ ] context.md Next Actions written
- [ ] decisions.md updated with key decisions
- [ ] ADR promoted to knowledge/ if decision is organizational
- [ ] Research findings in research.md (if any)

## Domain Boundaries

- [ ] Did NOT write production code
- [ ] Did NOT rewrite business requirements
- [ ] Did NOT implement — designed only

## Workspace

- [ ] Run `validate.sh` on workspace — PASS

## Handoff

- [ ] Engineering can understand what to build without asking
- [ ] Next Actions clearly states who should work next
