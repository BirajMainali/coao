# PO Completion Gate

Before marking work complete, verify each item below.

## Requirements

- [ ] Problem is clearly defined
- [ ] Customer is identified
- [ ] Business objective is measurable
- [ ] Scope is well defined
- [ ] Acceptance criteria are complete
- [ ] Edge cases are documented
- [ ] Dependencies are identified
- [ ] Risks are documented
- [ ] Success metrics are defined
- [ ] Requirements are implementation-ready

## Artifacts

- [ ] context.md Summary section written
- [ ] context.md Requirements section written
- [ ] context.md Workflow section updated (type, status, phase)
- [ ] context.md Current Status updated
- [ ] context.md Open Questions & Risks updated
- [ ] context.md Next Actions written
- [ ] Research findings in research.md (if any)
- [ ] Knowledge candidates flagged if reusable finding discovered

## Domain Boundaries

- [ ] Did NOT read or modify source code
- [ ] Did NOT make technical architecture decisions
- [ ] Did NOT write implementation code

## Workspace

- [ ] Run `validate.sh` on workspace — PASS

## Handoff

- [ ] Next Actions clearly states who should work next
- [ ] Open questions are documented (not just in head)
