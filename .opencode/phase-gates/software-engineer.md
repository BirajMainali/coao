# SE Completion Gate

Before marking work complete, verify each item below.

## Implementation

- [ ] Requirements are fully implemented
- [ ] Technical design has been followed
- [ ] Existing code conventions are respected
- [ ] No unnecessary complexity introduced
- [ ] No dead code or commented-out code
- [ ] No secrets, keys, or credentials in code
- [ ] Error handling implemented
- [ ] Edge cases considered
- [ ] Logging/observability appropriate

## Testing

- [ ] Unit tests pass
- [ ] Integration tests pass (if applicable)
- [ ] New code has test coverage
- [ ] Regression risk considered

## Code Quality

- [ ] No lint errors
- [ ] No type errors
- [ ] Abstractions justified, not premature
- [ ] Functions/modules have single responsibility
- [ ] No duplicate logic

## Artifacts

- [ ] context.md Implementation Summary written
- [ ] context.md Workflow section updated
- [ ] context.md Current Status updated
- [ ] context.md Open Questions & Risks updated
- [ ] context.md Next Actions written
- [ ] Implementation decisions in decisions.md (if significant)

## Domain Boundaries

- [ ] Did NOT change business requirements
- [ ] Did NOT redesign architecture without consulting SA
- [ ] Did NOT modify unrelated functionality

## Workspace

- [ ] Run `validate.sh` on workspace — PASS

## Handoff

- [ ] Next Actions clearly states who should work next (QA for review, or merge)
- [ ] Known issues documented, not hidden
