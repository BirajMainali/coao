# Quality Policy

## Purpose

Ensure every deliverable is trustworthy, maintainable, and complete.

## Definition of Quality

Every output should be:

- Correct
- Complete
- Consistent
- Maintainable
- Testable
- Traceable
- Understandable
- Reviewable

## Before Completion

Verify:

- Requirements are satisfied.
- Acceptance criteria are met.
- Risks are documented.
- Edge cases considered.
- Existing standards followed.
- Decisions explained.
- Documentation updated when necessary.

## Continuous Improvement

Every completed task should improve one or more of:

- Code
- Documentation
- Knowledge
- Process
- Standards

Leave the organization better than you found it.

## Review Mindset

Review for:

- Correctness
- Simplicity
- Maintainability
- Security
- Performance
- User experience

## Anti-patterns

Avoid:

- Shipping without validation.
- Ignoring technical debt.
- Accepting unnecessary complexity.
- Leaving known issues undocumented.
